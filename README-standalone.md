# My Routine - スタンドアロン版

認証機能付きの習慣管理Webアプリケーション（Node.js不要）

## 🌟 新機能：認証システム

### 追加された機能
- **ユーザー登録・ログイン**: 個人別のアカウント管理
- **セキュアなデータ保存**: ユーザー別のルーティン管理
- **パスワード保護**: 基本的なセキュリティ機能
- **セッション管理**: ログイン状態の保持

## 🚀 特徴

- **完全スタンドアロン**: サーバー不要、Node.js不要
- **認証機能**: ユーザー登録・ログイン・ログアウト
- **習慣管理**: 毎日・毎週・毎月のルーティン管理
- **進捗追跡**: 完了率の統計表示
- **フィルター機能**: 頻度別のルーティン表示
- **レスポンシブデザイン**: モバイル・デスクトップ対応
- **美しいUI**: モダンなデザインとアニメーション

## 📁 ファイル構成

```
My routine/

├── styles.css          # スタイルシート
├── script.js           # JavaScript（認証・ルーティン管理）
└── README-standalone.md # このファイル
```

## 🛠️ 使用方法

### 1. ローカルでの実行
1. `index.html` をブラウザで開く
2. 初回は会員登録画面が表示されます
3. ユーザー名、メールアドレス、パスワードを入力して登録
4. ログイン後、ルーティンの管理を開始

### 2. GitHub Pagesでの公開
1. GitHubで新しいリポジトリを作成
2. 以下のファイルをアップロード：
   - `index.html`
   - `styles.css`
   - `script.js`
3. リポジトリの設定でGitHub Pagesを有効化
4. `https://[ユーザー名].github.io/[リポジトリ名]` でアクセス可能

## 🔐 認証機能の詳細

### ユーザー登録
- **ユーザー名**: 一意のユーザー名（必須）
- **メールアドレス**: 連絡用（必須）
- **パスワード**: 6文字以上（必須）
- **パスワード確認**: 一致確認（必須）

### ログイン
- 登録したユーザー名とパスワードでログイン
- ログイン状態はブラウザに保存
- ログアウトでセッション終了

### データ管理
- 各ユーザーのルーティンは個別に保存
- localStorageを使用したクライアントサイド保存
- ブラウザのデータを削除するとデータも削除

## 🚀 デプロイ方法

### GitHub Pages
1. GitHubでリポジトリを作成
2. ファイルをアップロード
3. Settings > Pages でソースを選択
4. 公開URLを取得

### Netlify
1. Netlifyにサインアップ
2. ファイルをドラッグ&ドロップ
3. 自動的にデプロイ完了

### Vercel
1. Vercelにサインアップ
2. GitHubリポジトリを連携
3. 自動デプロイ設定
